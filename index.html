Frequently Asked Questions

1. The difference between var, let, and const
var: Function-scoped, can be re-assigned and re-declared. let: Block-scoped, can be re-assigned, but not re-declared. const: Block-scoped, cannot be re-assigned or re-declared.
2. The difference between map(), forEach(), and filter()
map(): Creates a new array by applying a function to each element. forEach(): Executes a function on each element, but does not return anything. filter(): Creates a new array with elements that pass the condition specified in the function.
3. Explain arrow functions and how they are different from regular functions
Arrow function: A shorter syntax for defining functions. It does not have its own this — it inherits this from the surrounding context. Arrow functions cannot be used as constructors and do not have the arguments object. Regular function: Uses the function keyword and has its own this. Regular functions can be used as constructors and have access to the arguments object.
4. How JavaScript Promises work
JavaScript Promises represent a value that may be available now, or in the future, or never. They work in three states: Pending: The promise is still processing. Resolved (Fulfilled): The promise has been successfully completed, and a result is available. Rejected: The promise failed, and an error is returned.
5. How closures work in JavaScript
A closure is a function that retains access to variables from its lexical scope, even after the outer function has finished executing. It allows inner functions to "remember" and manipulate variables from the outer function. Closures are commonly used to create private variables and maintain state in asynchronous programming.
